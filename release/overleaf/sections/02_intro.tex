\section{Introduction}
\vspace{-2pt}


% We propose that \emph{autonomous multi-agent evolution is a promising paradigm for open-ended discovery}. 

Many important scientific problems do not come with ground-truth answers~\citep{mang2025frontiercs}. What is the best heuristic for a logistics problem~\citep{chen2025heurigym, zheng2026learning}? How should one write the most efficient kernel~\citep{ouyangkernelbench}? In these settings, the objective is clear, but the optimal solution is unknown. As a result, one-shot generation is insufficient. Strong solutions must be discovered through iterative proposal, testing, revision, and progress over time. 

Recent advances in LLM-powered agents have made this paradigm increasingly effective. Systems such as FunSearch~\citep{romera2024funsearch}, AlphaEvolve~\citep{novikov2025alphaevolve}, and more~\citep{lange2025shinkaevolve,agrawal2025gepa,cemri2026adaevolve} show that LLMs can be embedded in evaluator-guided evolutionary search loops for open-ended discovery. Rather than attempting to solve the problem in a single pass, these methods place the LLM inside an outer-loop search procedure: the model proposes candidate programs conditioned on previously high-scoring solutions, external evaluators execute and score these candidates under task-specific objectives, and a predetermined evolutionary algorithm governs parent selection and population updates. This type of LLM-based evolution has proven effective across mathematical discovery, algorithm design, and systems optimization tasks.

\input{figures/01_paradigm}



However, current progress has largely been achieved with fixed evolutionary search, where key search decisions are independent of the agents, including which parent solutions to inspect and build on, when to run intermediate tests, and what knowledge to externalize and reuse later. For challenging open-ended problems, these choices are integral to the evolutionary algorithm and can substantially affect performance. This naturally leads to the following question: \textbf{Can stronger performance emerge when a greater part of the evolutionary algorithm is delegated to autonomous agents?}

The rigidity of these strategies is further amplified in multi-agent systems. Much of the existing literature relies on \emph{vertical scaling}: humans decompose the task, assign specialized roles, and define a fixed communication structure~\citep{hong2024metagpt,li2023camel,wu2024autogen}. While this paradigm has produced strong systems such as Sakana AI's AI Scientist~\citep{lu2024aiscientist} and Google's AI Co-Scientist~\citep{gottweis2025towards}, it assumes that the optimal decomposition and interaction topology are known in advance. For open-ended problems, that assumption is restrictive. This raises a further question: \textbf{Can multiple autonomous agents scale more effectively through \emph{horizontal parallelism}, by exploring in parallel, exchanging discoveries, and building on each other's progress over time?} Figure~\ref{fig:paradigm_comparison} illustrates the progression across these three paradigms.

% To study this setting, we introduce \method{}, a framework that allows autonomous agents to run long-horizon self-evolution loops toward increasingly better solutions. At each step, an agent first uses \method{} APIs to inspect prior attempts and shared knowledge, including historical solutions, distilled notes, and reusable skills. Based on this retrieved context, the agent reasons about the current search state, modifies the candidate solution, and may optionally run local tests before submitting the new version for evaluation. Evaluations are mediated through \method{} APIs while the evaluator itself remains isolated from the agents, reducing opportunities for reward hacking. As search proceeds, agents may also externalize observations as notes and package useful procedures, scripts, or tools as reusable skills. In the multi-agent setting, all agents share access to this evolving knowledge base: they can inspect one another's attempts, read and write notes, reuse skills, and thereby coordinate indirectly through persistent shared artifacts rather than a fixed communication protocol. To support long-horizon operation, \method{} further introduces heartbeat mechanisms---external interventions that periodically prompt reflection, consolidation, or redirection when agents become stuck or overly committed to a local context. Finally, each agent runs in an isolated workspace under manager-level orchestration, which preserves reproducibility across attempts and enables persistent execution over extended runs.

As a key step towards this new paradigm, we introduce \method, a framework for autonomous multi-agent evolution on open-ended problems. \method\ shifts decision-making from fixed algorithms to the agents themselves, supported by a shared persistent memory for continuous evolution. Agents iteratively refine solutions by retrieving, contributing, and distilling knowledge in the form of notes and reusable skills into a collective repository. By incorporating heartbeat mechanisms for periodic reflection and redirection, \method\ ensures robust exploration and knowledge accumulation, providing a task-agnostic architecture compatible with diverse agent implementations.

Empirically, CORAL establishes new SOTA on $8$ of $11$ tasks in mathematical and systems optimization, with a $2.5\times$ higher improvement rate and $10\times$ fewer evaluations than fixed evolutionary search baselines. On the stress-test Kernel Engineering task, four co-evolving agents push the score from $1,363$ to $1,103$ cycles (a $20\%$ gain), surpassing the previous best result. Ablation studies confirm that both agent autonomy and multi-agent evolution contribute to these gains.

Our contributions are threefold. First, we formulate autonomous evolution as a distinct paradigm for open-ended discovery and distinguish autonomous single-agent and multi-agent evolution from prior fixed evolutionary search. Second, we introduce CORAL, a framework that realizes this paradigm through shared persistent memory, asynchronous multi-agent organization, and heartbeat-based interventions for long-horizon search. Third, across mathematical, algorithmic, and systems optimization tasks, we show that CORAL substantially outperforms fixed evolutionary search baselines, and ablations and trajectory analyses show the importance of knowledge accumulation and multi-agent evolution.

% In studying this regime, we encountered a recurring and surprising pattern. Advanced agents often achieve higher ceilings when given more autonomy and less rigid structure. Compared to tightly constrained evolutionary setups such as [AlphaEvolve](https://deepmind.google/discover/blog/alphaevolve-a-gemini-powered-coding-agent-for-designing-advanced-algorithms/) and [OpenEvolve](https://github.com/algorithmicsuperintelligence/openevolve), we found that agents given greater autonomy to explore, reflect, and iterate often improve faster, reach stronger limits, and succeed more frequently. For example, on the [Erdős Min Overlap problem](https://en.wikipedia.org/wiki/Minimum_overlap_problem), using the same backbone model, Opus 4.6 without internet access, our autonomous setup achieves a 2.5× higher improved attempt rate than OpenEvolve, reaches 99% of state of the art performance roughly 10× faster with 7× fewer evaluation calls, and ultimately attains a better final score.

% This observation pushed us to build **CORAL**, an infrastructure for robust autonomous evolution. **CORAL** is designed to let agents fully leverage their autonomy while remaining reliable over long running searches. It provides isolated workspaces and separated evaluation to prevent reward hacking, session storage with automatic resume for sustained runs, a heartbeat mechanism for reflection and knowledge accumulation, infrastructure to support multi-agent evolution, and flexible task interfaces for any domain where candidate solutions can be generated and compared.

% Once **CORAL** was in place, we were able to go beyond single agent evolution and study multi-agent evolution. What we found was even more striking. On Anthropic's take-home task for a kernel engineer role, again without internet access, a single agent improved the state of the art from 1,363 cycles to 1,350, while a population of four agents pushed it dramatically further to 1,103.

% LLMs have made remarkable strides in advancing intelligence, with applications such as coding and solving math Olympiad problems. However, most of its use remains in the domain of accomplishing tasks that are known to be solvable. Recently, there is a growing interest in discovering whether LLMs can be applied to push the frontiers of human knowledge and solve complex problems without known solutions and engineer products that have never been previously built. In this regard, a bulk of research focuses on evolution-based strategies that iteratively improve the quality of a LLM's solution over multiple evolution steps. However, a critical challenge (as mentioned in ShinkaEvolve) is substantial sample inefficiency in the number of evolution steps required. Another challenge is the need to inject various ``structures'' to the evolution systems to validate the evolution result (DGM/HGM) and balance exploration against exploitation (ShinkaEvolve).

% While theoretically feasible, these challenges render real-life deployment of scalable evolution systems painstakingly difficult. For one, many of the evolution steps turn out to be wasteful (or dead nodes in the evolution tree). Works that attempt to address this issue inject complicated pipelines such as adjusting the LLM's temperature (ShinkaEvolve), prompt sampling (OpenEvolve), and research idea verification (AlphaResearch). Instead, following the philosophy of ``less structure, more intelligence'', we advocate for replacing all these optimization-level structures with an autonomous agent who (at each evolution step) analyzes the current state of the evolution, proposes research ideas, conducts experiments to self-verify the ideas, and present a summary documenting the findings.

% Although autonomous agent tends to produce more rigorous analyses of the evolution problem and provide better insights, they often take more time in each evolution step. To address this problem and make evolution really scalable, we propose a novel global communication mechanism that supports asynchronous evolution of multiple autonomous agents at the same time. Intuitively, such a global communication mechanism is somewhat akin to a research community comprised of many independent researchers. Each researcher references the current literature (i.e., existing evolution results) and proposes novel ideas under a pre-defined research topic.

% From a grander perspective, the progression of evolutionary intelligence has been through three stages. The inception of evolutionary intelligence lies in traditional genetic and evolutionary algorithms and culminates in the success of AlphaGo. The second stage is exemplified by the utilization of LLMs in algorithm evolution. Instead of the parametric updates heavily relied on by traditional methods, LLMs can directly update the algorithm, allowing for much greater flexibility in the evolution. The third and the latest stage sees evolutionary strategies such as dGM and OpenClaw that enable the LLM to modify the entire program, significantly expanding the problem sets that can be feasibly evolved. Coral sits at the fourth stage, where we offer autonomy to each evolution step, treating each step as an independent agent capable of developing its own thoughts and organizing its own optimization. Compared to previous stages, the autonomy of agent-ization of each step substantially injects more diversity and allows collaboration between agents to be more flexible. For example, autonomous agents can independently explore the codebase and optimization logs developed by other agents to gain insights that can only be obtained through rigidly hardcoded logic. Beyond this point, we envisage the next stage as an ``organization'' of agents, in which autonomous agents form (and potentially create) departments with heterogeneous missions that collectively contribute to a shared ultimate goal. We believe that our self-evolving framework using autonomous agents paves the way towards it.
