## Boundaries and comparison with arXiv 2505.22954

The DGM round-1 report caught P2, P10, P12, and P22 via compilation, but
missed P15 because its broad year exception was unscoped. This ARIS run differs
in four material ways:

- P2 is a real new leaf-check boundary: ARIS's only numeric claims are
  `supporting`, so explicit numeric rebinding is not checked. The DGM
  fixture used a core numeric claim and therefore fired.
- P10 confirms the same registered-float enforcement, while making a sharper
  scope distinction: ARIS's inline, label-free hero has no figure-index
  contract. The registered ARIS architecture figure is caught cleanly.
- P12 has no exact deletion fixture because every ARIS table is truthfully
  qualitative. Its verified-table equivalent still fires, preserving evidence
  that the numeric-binding gate activates when a table asserts verified status.
- P20 now fires by default because this case includes the later
  `physical_validation.enabled: true` template hardening; the older DGM report
  recorded the then-default inactive physical check. Conversely P22 is more
  serious here: removing the ARIS copied venue package is silent both to the
  semantic gate and real compilation, whereas DGM compilation caught its
  missing preamble input.

P9 remains an expected structural-validation limitation, P24/P25 remain
correct non-failures, and P15 remains concrete case ledger debt until the
unscoped `2026` exception is narrowed. The unavailable upstream profile could
only affect the P2 interpretation; no profile claim is made for any other
result.
