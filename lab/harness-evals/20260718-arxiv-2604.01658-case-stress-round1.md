### 最值得做持久修复的 harness-gap 候选

1. **P3 —— 数字注册表 `value` 与 `display_value` 不做交叉校验(新发现,超出已知 arxiv-flat gap)。**
   `check_number_value_consistency()`(`scripts/paper_harness_checks.py:3897`)把 `value` 与
   `display_value` 都塞进同一个 `direct_values` 列表,只与嵌套 `display` 字典比较。因此顶层
   `value: 1400` 与 `display_value: "1363"` 直接矛盾时静默通过。已注册数字的内部自洽性没有被守住,
   这是本轮唯一超出已知问题的、可复现的 harness 缺陷。**建议开 fix 分支**:让 `value` /
   `display_value` / `reported_value` 之间也走 `numeric_values_compatible()`,并加负向回归测试。
2. **P29 —— `release/arxiv-flat` 无法编译(已知 gap,复现确认)。** 见 baseline 报告:
   `compute_flatten_bundle()` 把 `style/*.sty|*.bst` 拍平到 flat 根,但
   `rewrite_flatten_asset_paths()` 不改写 `\usepackage{style/...}` /
   `\bibliographystyle{style/...}` 的 `style/` 前缀,导致 `! LaTeX Error: File
   'style/ccfa-paper.sty' not found.`。模板级、影响所有 case。
3. **P5 —— result-index 的 `numeric_ids` 反向绑定未被强制(次要)。** 数字→结果的反向一致性检查只认
   result 条目的 `numbers` 字段,而本模板用的是 `numeric_ids`,因此从 result-index 删除
   `numeric_ids` 不触发任何错误。数字→结果方向仍由 `result_ids` 表达,严重度低,但属于校验盲区。

  这是豁免的已知代价,promote 出 fixture 前需补真实 kit 验证。
